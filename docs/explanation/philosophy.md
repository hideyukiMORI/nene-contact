# Philosophy

- **Small surface, clear SSOT** — submissions live in Contact; siblings get copies via HTTP, not shared tables.
- **Embed-first** — the product is successful when the script line works on a static site without Records or Concierge.
- **Human sends, AI assists** — MCP reads and proposes; outbound email from AI is gated and audited.
- **Fail open for operators** — if Invoice handoff fails, the submission stays in Contact with a retry action.
- **Same governance as siblings** — Issue-driven, NENE2 runtime, OpenAPI contract, English docs.
