# 管理コンソール UI ガイド（ヘルプ）

NeNe Contact の管理コンソール（`/console/`）の各画面と、UI 要素の意味をまとめたヘルプです。
初めて触る方は [チュートリアル](./tutorial.md) と合わせてどうぞ。

> 管理コンソールは運用者（=データ管理者）向けの画面です。コンプライアンス上の責任範囲は
> [運用者ガイド](../operations/operator-guide.md)、用語は
> [terminology](../explanation/terminology.md) を参照してください。

## 画面の全体像

ログイン後、上部ナビに 4 つのセクションがあります。右端に **ログアウト**。

| ナビ | 役割 |
| --- | --- |
| **ダッシュボード** | ログイン中のアカウント情報・各機能への入口 |
| **フォーム** | お問い合わせフォームの一覧・作成・通知設定 |
| **受信箱** | 届いた送信の一覧・対応 |
| **ユーザー** | 組織内の操作アカウント管理 |

---

## 1. ログイン（`/console/login`）

| 項目 | 意味 |
| --- | --- |
| **メールアドレス / パスワード** | 運用者アカウントの資格情報。発行は [運用者ガイド](../operations/operator-guide.md) の `tools/create-user.php` |
| **ログイン** | 認証して管理コンソールに入ります（失敗時は「メールアドレスまたはパスワードが正しくありません。」） |

ログインは JWT（メモリ保持）です。再読み込みすると再ログインが必要になります。

## 2. ダッシュボード（`/console/`）

ログイン中のアカウント（メール・ロール）を表示する入口画面です。実際の作業は上部ナビから
各セクションへ移動します。

## 3. フォーム（`/console/contact-forms`）

お問い合わせフォームの一覧です。

| 列 | 意味 |
| --- | --- |
| **名前** | フォームの表示名（管理用） |
| **公開キー**（`public_form_key`） | サイト埋め込み（配信）で使う公開識別子。内部 ID とは別物 |
| **言語** | フォームが対応するロケール（`ja` / `en`） |
| **状態** | `active` などの公開状態 |
| **チャネル** | そのフォームの [通知チャネル設定](#4-通知チャネルcontact-formsidchannels) へ |

右上の **新規フォーム** から作成に進みます。

### 3.1 テンプレートを選択（`/console/contact-forms/new`）

新規作成は、まず **テンプレートを選択**します。用途に近いものから始められ、あとから自由に編集
できます。

- **空から作成** … 白紙のフォーム
- **お問い合わせ / 資料請求 / 予約・相談 / 採用応募 / サポート依頼** … 代表的な項目があらかじめ
  入った雛形

> テンプレートは便利機能であると同時に「正しい初期設定」の型でもあります。各テンプレートには
> **ハニーポット**が入り、個人情報を集めるものは **同意必須が既定 ON** です。マイナンバーや
> カード番号などの禁止項目は、そもそも項目タイプに存在しません（憲章 §8）。

「このテンプレートを使う」を押すと、項目入りのビルダーが開きます（**テンプレート選択に戻る**で
やり直せます）。

### 3.2 フォームビルダー

| UI | 意味 |
| --- | --- |
| **フォーム名** | 管理用の名前 |
| **対応言語** | 提供するロケール（ja/en）。**既定**は初期表示言語 |
| **同意 / 同意を必須にする** | チェックすると送信時に同意が必須に。**同意文言（ja/en）** を設定（憲章 §3） |
| **フィールドを追加** | 下記の項目タイプを追加（パレット） |
| **フィールド名** | 送信データのキー（英数）。例: `email` |
| **ラベル（ja/en）** | 訪問者に表示するラベル |
| **必須** | 入力必須にする |
| **選択肢（1 行に 1 つ）** | `選択` タイプのときの候補 |
| **並べ替え** | ドラッグで項目順を変更 |
| **保持日数**（`retention_days`） | この日数を過ぎた送信は自動で削除対象に（個人情報は所定の猶予後に消去、ADR 0016 / 憲章 §5） |
| **許可オリジン（1 行に 1 つ）** | このフォームを埋め込んでよいサイトのオリジン許可リスト（空なら制限なし、ADR 0010） |
| **作成** | フォームを保存（名前と 1 つ以上のフィールドが必要） |

### 3.3 項目タイプの意味

| タイプ | 用途 |
| --- | --- |
| **テキスト** | 1 行の自由入力（氏名・会社名など） |
| **メール** | メール形式を検証する入力 |
| **複数行テキスト** | 問い合わせ内容など長文 |
| **選択** | 候補から 1 つ選ぶ（優先度など） |
| **チェックボックス** | はい/いいえ |
| **ファイル** | 添付ファイル（容量・形式制限あり） |
| **ハニーポット** | 訪問者には見えないスパム対策項目。値が入っていれば静かに破棄（ADR 0010） |

## 4. 通知チャネル（`/contact-forms/{id}/channels`）

フォームに送信が届いたときの **通知先**を設定します（トランザクション通知。マーケ配信ではあり
ません、憲章 §7）。

| 種別 | 主な設定 |
| --- | --- |
| **メール** | 送信先メールアドレス |
| **Slack** | Webhook URL |
| **Chatwork** | API トークン / ルーム ID |
| **Webhook** | URL / 署名シークレット（HMAC 署名付き送信） |

**有効** 列で稼働状態を確認できます。シークレット類は保存時に暗号化され、画面・通知・ログに生値
は出ません（憲章 §6）。

## 5. 受信箱（`/console/submissions`）

届いた送信の一覧です。

| 列 | 意味 |
| --- | --- |
| **ID** | 送信の識別子 |
| **フォーム** | どのフォームからか |
| **状態** | 対応状況（下記） |
| **受信日時** | 届いた日時 |

下部の **前へ / 次へ** でページ送り。行をひらくと詳細（`/console/submissions/{id}`）へ。

### 5.1 送信詳細

| UI | 意味 |
| --- | --- |
| **状態** | **未対応 → 対応中 → 解決済み**（迷惑なものは **スパム**）。変更は監査に記録されます |
| **項目 / 値** | 訪問者が入力した内容 |
| **同意日時** | 同意必須フォームでの同意の証跡（憲章 §3） |
| **メモ** | 運用者の対応メモを追加（監査対象） |

> IP / ユーザーエージェントは不正調査目的のみで保持し、既定では表示・書き出ししません（憲章 §2）。
> 送信の閲覧・書き出し・状態変更・削除・訂正はすべて監査ログに残ります（ADR 0013）。物理削除は
> 行わず、個人情報はその場で消去します（ADR 0016）。

## 6. ユーザー（`/console/users`）

組織内の操作アカウントを管理します。

| UI | 意味 |
| --- | --- |
| 一覧（メール / ロール / 状態） | 既存アカウント |
| **ユーザーを追加** | メール・パスワード・ロールで作成 |
| ロール | **スーパー管理者 / 管理者 / 編集者**（権限の範囲が異なる） |
| 状態 | **有効 / 無効** |

自分自身のロール・状態は変更できません（誤操作防止）。

## 7. 配信（サイトへの埋め込み）

作成したフォームは、サイトに 1 行のスクリプトを貼って公開します。**公開キー**（フォーム一覧で
確認）を `data-form` に指定します。

```html
<script src="https://YOUR_HOST/embed.js" data-form="公開キー" data-trigger="floating" data-lang="ja" async></script>
```

| 属性 | 意味 |
| --- | --- |
| `data-form` | フォームの **公開キー**（必須） |
| `data-trigger` | `floating`（右下に浮く起動ボタン・既定）/ `button`（ボタン設置）/ `inline`（その場に表示） |
| `data-button-label` | `button` のときのボタン文言 |
| `data-lang` | 表示言語（`ja` / `en`） |

- 本番は **HTTPS 必須**（憲章 §6）。埋め込み先は **許可オリジン**に含めてください（3.2 参照）。
- 動作確認用に `/embed-demo.html?form=公開キー` が使えます。
