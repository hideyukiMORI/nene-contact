# チュートリアル：はじめての問い合わせフォーム

管理コンソールに**ログイン**してから、フォームを**作成**し、サイトへ**配信**し、実際に**表示**して
**結果を受信箱で確認**し、**ログアウト**するまでを、ひとりのペルソナになりきって通します。
各画面の意味は [UI ガイド](./admin-ui-guide.md) を参照してください。

---

## 登場人物（ペルソナ）

> **田中 美咲（たなか みさき）／そら設計事務所 Web 担当**
> 小さな設計事務所のサイト運用を一人で担当。コードは少し読める程度。
> 「サイトに『お問い合わせ』を置いて、メールで届くようにしたい。難しい設定はしたくない。」

**前提**

- 管理者アカウントが発行済み（[運用者ガイド](../operations/operator-guide.md) の `tools/create-user.php`）。
  例: `misaki@sora-arch.example` / パスワードは発行済み。
- サイトは HTTPS で公開済み（例: `https://sora-arch.example`）。
- NeNe Contact は `https://contact.sora-arch.example`（自社ホスト）で稼働。

---

## ステップ 1. ログイン

1. `https://contact.sora-arch.example/console/` を開く。
2. **メールアドレス**に `misaki@sora-arch.example`、**パスワード**を入力して **ログイン**。

✅ **結果**：ダッシュボードが開き、上部ナビ（ダッシュボード / フォーム / 受信箱 / ユーザー）が
表示される。

## ステップ 2. フォームを作成（テンプレートから）

1. ナビ **フォーム** → 右上 **新規フォーム**。
2. **テンプレートを選択**で **お問い合わせ** の「このテンプレートを使う」を押す。
   - お名前 / メールアドレス / お問い合わせ内容 と、スパム対策のハニーポットが入った状態で
     ビルダーが開く。**同意を必須にする**は既定で ON。
3. **フォーム名**に「お問い合わせ」と入力。**対応言語**は `ja`（必要なら `en` も）。
4. （任意）**許可オリジン**に `https://sora-arch.example` を 1 行で入力（埋め込み先を限定）。
5. **作成** を押す。

✅ **結果**：フォーム一覧に「お問い合わせ」が追加され、**公開キー**（例: `8f3c…`）が表示される。

> 困ったら：**作成**が押せないときは「フォーム名」と「1 つ以上のフィールド」が必要です。

## ステップ 3. 通知先（メール）を設定

1. 一覧の「お問い合わせ」行の **チャネル** を開く。
2. **チャネルを追加** → 種別 **メール** → **送信先メールアドレス**に `info@sora-arch.example` を入力
   → **チャネルを追加**。

✅ **結果**：チャネル一覧にメール通知が **有効** で並ぶ。送信が届くとこのアドレスに通知される。

## ステップ 4. 公開キーを控える

1. ナビ **フォーム** に戻り、「お問い合わせ」行の **公開キー**（`<code>` 表示）をコピー。

✅ **結果**：配信に使うキー（例: `8f3c1a…`）が手元に揃う。

## ステップ 5. サイトへ配信（埋め込み）

1. サイトのテンプレート（全ページ共通フッターなど）に、次の 1 行を貼る。`data-form` に控えた
   公開キーを入れる。

   ```html
   <script src="https://contact.sora-arch.example/embed.js"
           data-form="8f3c1a…" data-trigger="floating" data-lang="ja" async></script>
   ```

2. サイトを再デプロイ／反映する。

✅ **結果**：埋め込んだページの右下に、問い合わせ用の **起動ボタン（floating）** が出る。

> 補足：その場に表示したいなら `data-trigger="inline"`、ボタン設置なら `data-trigger="button"`。
> 本番は **HTTPS 必須**、埋め込み先は **許可オリジン**に含めること（[UI ガイド 7](./admin-ui-guide.md#7-配信サイトへの埋め込み)）。

## ステップ 6. 表示してテスト送信

1. 自社サイト `https://sora-arch.example` を開き、右下の起動ボタンをクリック。
2. フォームが開くので、お名前・メール・内容を入力し、**同意**にチェックして送信。

✅ **結果**：送信完了。設定したメールアドレス（`info@…`）に通知が届く。

> 動作確認だけしたいときは、サイトに貼る前に `…/embed-demo.html?form=公開キー` でも試せます。

## ステップ 7. 受信箱で結果を確認・対応

1. 管理コンソールのナビ **受信箱**。
2. 一覧に新着（ID / フォーム / 状態 **未対応** / 受信日時）が並ぶ。行を開く。
3. 詳細で **項目 / 値**（入力内容）と **同意日時** を確認。
4. **状態**を **対応中** → 折り返し連絡後に **解決済み** へ変更。必要なら **メモ**に対応記録を残す。

✅ **結果**：問い合わせを 1 件、受信→確認→対応まで完了。状態変更・メモ・閲覧はすべて監査に
記録される（ADR 0013）。

> 個人情報の扱い：IP / UA は既定で非表示、削除は物理削除せず個人情報をその場で消去（ADR 0016）。
> 本人からの開示・訂正・削除の依頼対応は [運用者ガイド 6](../operations/operator-guide.md#6-data-subject-rights--retention)。

## ステップ 8. ログアウト

1. 上部ナビ右端の **ログアウト**。

✅ **結果**：ログイン画面に戻る。トークンは破棄され、再操作には再ログインが必要。

---

## まとめ（このチュートリアルでやったこと）

ログイン → テンプレートでフォーム作成 → メール通知設定 → 公開キー取得 → サイトへ 1 行で配信 →
表示してテスト送信 → 受信箱で結果確認・対応 → ログアウト。
